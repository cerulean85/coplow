# COPLOW (COoperator + PLan + flOW)

**AI Agent Swarm Powered Ultra-Fast Service Planning & System Design Framework**

Input just **one initial concept (prompt)**, and the Orchestrator (PMO) leading **10 core agents + 4 on-demand specialists** will collaborate to deliver a single, complete **“Service Planning and System Design Document”**.

---

## ✨ Core Objectives

- Turn a raw idea into a full pipeline: **Business Strategy → Product Planning → UX/UI → Technical Architecture → Data/AI → Security/QA → Marketing/GTM** in one seamless flow
- Minimize human intervention while maintaining **agency-level quality**
- Systematically control token costs, hallucinations, and logical errors to rapidly produce **real business-viable deliverables**

---

## 🏗️ Agent Structure (10 + 4 On-demand)

### Core 10 Agents (Always Active)
1. **Orchestrator (PMO)** – Project Conductor
2. **Business Strategist**
3. **Product Manager (PM)**
4. **UX Designer**
5. **UI/Visual Designer** (New)
6. **System Architect**
7. **Data/AI Specialist**
8. **Security & Compliance**
9. **QA / Red Team Critic**
10. **Marketing / GTM Strategist**

### On-demand 4 Specialists (Spawned only when needed)
- Content/Copy Strategist
- DevOps / SRE
- Accessibility & Localization
- Cost & Resource Estimator

---

## 📍 4-Stage Workflow (Structured Pipeline)

1. **Stage 1: Discovery & Definition** (Sequential)  
   Concept → Business Model + User Persona → PRD + MVP

2. **Stage 2: Design & Architecture** (Parallel)  
   - Track A: UX/UI Design  
   - Track B: Architecture + Data/AI + DevOps

3. **Stage 3: Verification & Refinement** (Red Team)  
   Security/QA Review + Debate Round + Delta Update

4. **Stage 4: Go-To-Market & Final Merge**  
   GTM Strategy + Full Document Integration

---

## 🚀 Core Operating Principles (Swarm Health & Governance)

- **Human-in-the-Loop (HiL)**: User approval required at the end of each stage
- **Parallel Processing + Delta Update**: Minimize unnecessary rework
- **Debate Round**: Automatic logical debate before Orchestrator decision
- **Structured Output**: All results forced to JSON/Markdown templates (↓ hallucination, ↑ parsability)
- **Versioning & Traceability**: Version ID + Parent ID for perfect change history
- **Swarm Health Monitoring**: Real-time dashboard for token usage, conflict frequency, HiL rejection rate
- **Safety Mechanisms**: Human Escalation after 3 failures + Graceful Rollback

---

## 📦 Final Deliverables

Delivered as **one unified document**:

- Service Planning Document (PRD + BM + KPI)
- UX/UI Design (IA, User Flow, Design System JSON, Image Generation Prompts)
- System Architecture (Tech Stack, ERD, API Spec, Infrastructure)
- Data & AI Strategy
- Security & Compliance Report
- GTM & Marketing Strategy
- Cost Estimation Table, etc.

초기 컨셉(프롬프트) **하나만 입력하면**, 오케스트레이터(PMO)가 지휘하는 **10개 핵심 에이전트 + 4개 온디맨드 스페셜리스트**가 협업하여  
**“서비스 기획 및 시스템 설계서”** 하나로 완성된 산출물을 만들어 내는 시스템입니다.

## ✨ 핵심 목적

- 사용자가 아이디어만 던지면 → **비즈니스 전략 → 제품 기획 → UX/UI → 기술 아키텍처 → 데이터·AI → 보안·QA → 마케팅·GTM**까지 **전 과정을 한 번에** 처리
- 인간의 개입은 최소화하면서도 **품질은 전문 에이전시 수준**으로 유지
- 토큰 비용·환각·논리 오류를 체계적으로 통제하면서 **실제 사업화 가능한 결과물**을 빠르게 도출

## 📍 4단계 워크플로우 (Structured Pipeline)

1. **Stage 1: 발산과 정의** (순차)  
   컨셉 → BM + 타겟 페르소나 → PRD + MVP

2. **Stage 2: 구체화 및 설계** (병렬)  
   - Track A (UX/UI 디자인)  
   - Track B (아키텍처 + 데이터/AI + DevOps)

3. **Stage 3: 검증과 보완** (레드팀)  
   보안·QA 검토 + Debate Round + Delta Update

4. **Stage 4: 시장 진출 및 최종 병합**  
   GTM 전략 + 모든 산출물 통합 → 최종 설계서

## 🚀 핵심 운영 원칙 (Swarm Health & Governance)

- **Human-in-the-Loop (HiL)**: 각 Stage 끝에 사용자 승인 필수
- **병렬 처리 + Delta Update**: 불필요한 재작업 최소화
- **Debate Round**: 오류 발견 시 자동 토론 → 오케스트레이터 결단
- **Structured Output**: 모든 결과는 JSON/Markdown 템플릿 강제 (환각 ↓, 파싱 ↑)
- **Versioning & Traceability**: Version ID + Parent ID로 변경 이력 완벽 관리
- **Swarm Health 모니터링**: 토큰 사용량, 충돌 빈도, HiL 거절률 등 실시간 대시보드
- **안전장치**: 3회 실패 시 Human Escalation + Graceful Rollback

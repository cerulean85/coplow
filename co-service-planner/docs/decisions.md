### 5.2 RICE Scoring 기준
- **Reach**: 월간 영향받는 사용자 수
- **Impact**: 0.25 / 0.5 / 1 / 2 / 3
- **Confidence**: % (확신도)
- **Effort**: 개발 인월(person-month)
- **RICE Score** = (Reach × Impact × Confidence) / Effort

## 섹션 1. 서비스 컨셉 정의

### 핵심 한 줄 정의  
정신 노동에 종사하는 직장인이 기록한 내용을 통해 인지 오류를 인식하고 개선하여, 더 명확한 사고 과정을 갖도록 돕는 개인화된 프로그램.

### JTBD  
- IT 개발자: "업무 중 블로그에 기술 관련 글을 작성할 때, 나의 이해도를 높이기 위해 이 서비스를 사용한다."  
- 마케팅 전문가: "고객의 반응을 정확히 파악하기 위해 마케팅 캠페인 데이터를 분석할 때, 이 서비스를 사용한다."  
- 프리랜서 작가: "내 사고 과정을 명확히 하고 독자들에게 더 명확한 메시지를 전달하기 위해 글을 쓸 때, 이 서비스를 사용한다."

### 핵심 Pain Point  
1. 자신의 인지 오류를 인식하지 못해 불안감을 느낌 — 근거: 한국인 10명 중 9명이 인지적 오류 습관을 가지고 있음 (경향신문).  
2. 기록한 내용을 분석하는 데 시간이 많이 소요됨 — 근거: IT 개발자 김민수는 매일 1시간 이상을 블로그 글 작성 및 분석에 사용.  
3. 자신의 사고 과정이 명확하지 않아 창의적인 작업에 어려움을 겪음 — 근거: 프리랜서 작가 박수진은 글을 쓸 때 사고 과정이 명확하지 않아 메시지 전달에 어려움이 있음.

### 가치 제안  
사용자의 기록을 바탕으로 인지 오류를 실시간으로 분석하고 개인화된 피드백을 제공하여, 단순 정보 제공에 그치는 경쟁사와 차별화된 심층 분석 기능을 가진 서비스를 제공.

### USP  
1. 사용자가 작성한 기록을 실시간으로 분석하여 개인화된 인지 오류 피드백을 제공한다.  
2. 단순한 오류 인식에 그치지 않고, 사용자의 사고 패턴을 심층적으로 분석하여 장기적인 개선을 도모한다.  
3. 광고에 의존하지 않는 수익 모델을 채택하여, 사용자는 방해받지 않고 프로그램의 모든 기능을 활용할 수 있다.

## 섹션 4. User Flow & Information Architecture

### 4.1 핵심 User Flow — [회원가입/온보딩]  
[시작 화면] → [회원가입 화면] → (조건: 이메일/소셜 로그인 선택) → [이메일 입력 화면] 또는 [소셜 로그인 화면] → [온보딩 화면] → [프로필 설정 화면] → [완료 화면]

### 4.2 핵심 User Flow — [핵심 기능 사용]  
[대시보드 화면] → [기록 작성 화면] → (조건: 저장 전 오류 인식 여부) → [저장 화면] 또는 [오류 피드백 화면] → [기록 확인 화면] → [인지 오류 분석 결과 화면]  

### 4.3 핵심 User Flow — [결제/구독]  
[서비스 웹사이트] → [구독 옵션 선택 화면] → (조건: 무료 체험 후 결제 여부) → [결제 정보 입력 화면] 또는 [무료 버전 사용 화면] → [결제 완료 화면] → [구독 확인 화면]

### 4.4 Conditional Flow — [조건명]  
[기록 작성 화면] → (조건: 오류 인식 기능 활성화 여부) → [오류 피드백 화면] → (조건: 사용자 수정 여부) → [기록 수정 완료 화면] 또는 [기록 작성 화면] 

### 4.5 Error Flow — [에러 처리]  
[회원가입 화면] → (조건: 필수 입력 사항 미제출) → [오류 메시지 화면] → (조건: 사용자 재입력 여부) → [회원가입 화면] 또는 [앱 종료 화면]

### 4.6 정보 아키텍처 (IA)  
```
├── 홈
│   ├── 대시보드
│   ├── 기록 작성
│   ├── 인지 오류 분석
├── 회원가입
│   ├── 이메일 회원가입
│   ├── 소셜 로그인
│   └── 프로필 설정
├── 결제 및 구독
│   ├── 무료 체험
│   ├── 유료 구독 옵션
│   └── 결제 정보 입력
├── 도움말
│   ├── FAQ
│   ├── 고객 지원
│   └── 사용자 가이드
├── 계정 관리
│   ├── 프로필 수정
│   ├── 구독 관리
│   └── 로그아웃
```

| 채팅 또는 포럼 기능       | 커뮤니티 인터랙션은 초기 MVP 목표에서 벗어나며, 기능 구현과 운영 비용이 높아 서비스 핵심 가치와의 일치를 이루기 어려움. |
| AI 정서 분석 기능         | 초기 제품에 실질적인 가치를 더하는 데 더 많은 자원과 시간이 필요하며, 사용자 피드백 데이터가 충분하지 않아서 우선 적용되지 않음. |
| 다양한 언어 지원         | 초기 타겟 시장이 한국 내 사용자로 한정되므로, 여러 언어 지원은 불필요하며, 자원 낭비 우려가 있음. |

## 섹션 6. MVP 정의 & Phase별 로드맵

### MVP 핵심 기능 세트  
1. **기록 작성 기능** — 포함 이유: 사용자가 자신의 생각과 아이디어를 기록할 수 있는 기본 기능으로, 모든 서비스의 출발점이자 사용자 참여를 유도하는 핵심 요소입니다.  
2. **인지 오류 분석 기능** — 포함 이유: 기록된 내용을 기반으로 사용자가 인지 오류를 인식하고 피드백을 받을 수 있도록 하여, 핵심 가치 제안을 실현하는 데 필수적입니다.  
3. **실시간 피드백 기능** — 포함 이유: 사용자가 기록 후 즉시 피드백을 받아 개선할 수 있는 기회를 제공함으로써, 서비스의 효용성을 극대화합니다.  
4. **온보딩 및 프로필 설정** — 포함 이유: 신규 사용자들이 서비스에 쉽게 적응하고 사용할 수 있도록 지원하여, 초기 사용자 경험을 향상시키는 데 필수적입니다.  
5. **구독 관리 기능** — 포함 이유: 사용자가 구독 상태를 간편하게 관리할 수 있도록 하여, 오래 지속 가능한 사용을 유도하고 수익 모델의 안정성을 높입니다.

### 테스트 대상

### 검증 시나리오
1. **기록 작성 시나리오**: 사용자가 특정 주제에 대해 기록을 작성하고, 오류 인식 기능을 경험한다.
2. **인지 오류 분석 시나리오**: 사용자가 기록을 저장한 후, 분석 결과를 받아보는 과정을 확인한다.
3. **회원가입 및 프로필 설정 시나리오**: 신규 사용자가 회원가입 및 프로필 설정을 진행하면서 발생하는 문제를 확인한다.

### 성공 지표
- **태스크 완료율**: 각 시나리오에서 사용자가 지정된 태스크를 완료하는 비율 (목표: 80% 이상)
- **에러율**: 사용자가 각 태스크를 수행하는 동안 발생한 오류의 수 (목표: 5% 이하)
- **NPS (Net Promoter Score)**: 사용자의 만족도 및 추천 의향 조사 (목표: 50 이상)

## 섹션 4. User Flow & Information Architecture

### 4.1 핵심 User Flow — 회원가입/온보딩
[회원가입 화면] → [이용약관 동의 화면] → [기본 정보 입력 화면] → [계정 생성 완료 화면] → [온보딩 가이드 화면]

### 4.2 핵심 User Flow — 핵심 기능 사용
[메인 대시보드 화면] → [기록 작성 화면] → [기록 제출 화면] → [분석 결과 화면] →  (조건: 첫 사용자인가?) → [온보딩 피드백 화면] / [분석 결과 화면]

### 4.3 핵심 User Flow — 결제/구독
[구독 옵션 화면] → [결제 정보 입력 화면] → [결제 요청 화면] → [결제 완료 화면] → [구독 활성화 화면]

### 4.4 Conditional Flow — 첫 사용자의 경우
[분석 결과 화면] → (조건: 첫 사용자인가?) → [온보딩 피드백 화면] → [메인 대시보드 화면] 
(조건: 아니면) → [메인 대시보드 화면]

### 4.5 Error Flow — 오류 발생 처리 화면 
[기록 제출 화면] → (조건: 제출 실패) → [오류 메시지 화면] → [재시도 버튼 화면] 
(조건: 재시도를 원하는가?) → [기록 작성 화면] / [메인 대시보드 화면]

### 4.6 Information Architecture
```
├── 홈
│   ├── 메인 대시보드
│   ├── 기록 작성
│   ├── 분석 결과
│   └── 피드백
├── 회원가입
│   ├── 이용약관
│   ├── 기본 정보 입력
│   └── 계정 생성 완료
├── 결제
│   ├── 구독 옵션
│   ├── 결제 정보 입력
│   ├── 결제 요청
│   └── 결제 완료
├── 내 계정
│   ├── 프로필 설정
│   ├── 결제 정보 관리
│   └── 로그아웃
└── 고객 지원
    ├── FAQ
    ├── 문의하기
    └── 서비스 이용안내
```

이 설계는 사용자 Journey에서의 핵심 행동 단계에 대해 직관적으로 이해할 수 있도록 흐름도 및 메뉴 구조를 구성하였으며, 조건부 흐름과 에러 흐름을 통해 사용자의 다양한 경험을 포괄합니다.

## 섹션 6. MVP 정의 & Phase별 로드맵

### MVP 핵심 기능 세트
1. **실시간 인지 오류 분석** — 포함 이유: 사용자가 기록한 내용을 실시간으로 분석하여 인지 오류를 즉시 식별하여 문제 인식을 높이는 데 도움을 주기 때문에 필수적입니다.
2. **개인화된 피드백 제공** — 포함 이유: 분석 결과를 바탕으로 사용자에게 맞춤형 피드백을 제공함으로써 사용자의 자기 개선을 유도하는 핵심 기능으로, 사용자의 지속적인 참여를 도모합니다.
3. **기본 분석 결과 화면** — 포함 이유: 사용자가 자신의 기록에 대한 분석 결과를 쉽게 확인할 수 있는 화면을 제공함으로써 이해도를 높이고 서비스 활용도를 증대시키기 위해 요구됩니다.
4. **회원가입/온보딩 프로세스** — 포함 이유: 직관적이고 간편한 회원가입 및 온보딩 과정은 초기 사용자 경험을 최적화하여 이탈률을 최소화하기 위한 필수 요소입니다.
5. **사용 기록 저장 기능** — 포함 이유: 사용자가 작성한 기록을 안전하게 저장하는 기능은 사용자 신뢰도 향상과 서비스 지속 사용을 보장하기 위해 매우 중요합니다.

## 섹션 10. 사용자 검증 계획
### 테스트 대상
- **인원**: 15-20명
- **프로파일**:  IT 개발자, 마케팅 전문가, 프리랜서 작가 등 직장인 3명의 페르소나를 기반으로 선정.

### 검증 시나리오
1. 사용자 계정 생성 및 온보딩 과정 수행: 회원가입에서 시작해 자동으로 대시보드까지 이동.
2. 기록 작성 및 제출: 사용자가 생성한 기록을 제출하고 분석 결과를 확인.
3. 피드백 활용: 분석 결과를 기반으로 주어진 피드백을 사용하여 개인 기록을 개선.

### 성공 지표
- **태스크 완료율**: 각 사용자 시나리오의 목표 달성 여부.
- **에러율**: 사용자가 각 단계에서 마주친 오류의 수.
- **NPS**: 사용자 만족도 설문조사에서 얻은 평가.

## 섹션 1. 서비스 컨셉 정의

### 핵심 한 줄 정의
사용자가 작성한 프라이빗 기록을 기반으로 자신의 무지를 발견하고 사고 패턴을 개선할 수 있는 AI 어시스턴트 서비스.

### JTBD
- **IT 개발자 김민수**: "신기술을 배우는 과정에서 이해도를 높이기 위해 이 서비스를 사용한다."
- **마케팅 전문가 이지영**: "마케팅 캠페인을 분석할 때, 고객의 반응을 정확히 파악하기 위해 이 서비스를 사용한다."
- **프리랜서 작가 박수진**: "글을 쓸 때 사고 과정을 명확히 하고 독자에게 더 명확한 메시지를 전달하기 위해 이 서비스를 사용한다."

### 핵심 Pain Point
1. 자신의 인지 오류를 인식하지 못해 불안감을 느낌 — 근거: 한국인 10명 중 9명이 인지적 오류 습관을 가지고 있음 (경향신문).
2. 기록한 내용을 분석하는 데 시간이 많이 소요됨 — 근거: IT 개발자 김민수는 매일 1시간 이상을 블로그 글 작성 및 분석에 사용.
3. 잘못된 인지 오류로 인해 사회적 관계에서 오해가 발생 — 근거: 마케팅 전문가 이지영은 고객 피드백을 잘못 해석하여 캠페인 전략에 오류 발생.

### 가치 제안
심리적 엑스레이, 투명한 근거 제시 및 프롬프트의 예술적 은닉을 통해 사용자의 무의식적 사고 패턴을 분석하고 개선하는 독창적인 접근 방식으로, 기존의 단순 정보 제공이나 광고에 의존하는 서비스와 차별화된다.

### USP
1. 사용자가 작성한 기록을 바탕으로 실시간으로 인지 오류를 분석하고 개인화된 피드백을 제공하여 사고 과정을 명확히 하는 심층 분석 기능을 갖춘다.
2. 광고 없이 사용자 경험을 최우선으로 하여, 사용자에게 방해받지 않고 프로그램의 모든 기능을 활용할 수 있는 환경을 제공한다.
3. 데이터 기반 분석과 심리학적 접근을 결합하여 사용자가 자신의 사고 패턴과 편향을 효과적으로 이해하고 개선할 수 있도록 돕는다.

## 섹션 4. User Flow & Information Architecture

### 4.1 핵심 User Flow — 회원가입/온보딩
[홈 화면] → [가입 화면] → [이메일 입력 화면] → [가입 확인 화면] → [온보딩 설문 화면] → [서비스 소개 화면] → [대시보드 화면]

### 4.2 핵심 User Flow — 핵심 기능 사용
[대시보드 화면] → [기록 추가 화면] → [기록 입력 화면] → [기록 분석 화면 (결과)] → [피드백 확인 화면] → [기록 관리 화면]

### 4.3 핵심 User Flow — 결제/구독
[홈 화면] → [구매 화면] → [결제 정보 입력 화면] → [결제 확인 화면] → [영수증 화면] → [대시보드 화면]

### 4.3 Conditional Flow — 온보딩 설문 작성 여부
[온보딩 설문 화면] → (조건: 설문 작성 완료 여부) → [서비스 소개 화면] (Y) / [온보딩 설문 화면 (재작성 요청)] (N)

### 4.4 Error Flow — 기록 분석 오류
[기록 추가 화면] → (조건: 입력 오류 발생) → [입력 오류 화면] → [오류 수정 안내 화면] → [기록 입력 화면] → [기록 분석 화면 (결과)]

### 4.5 Information Architecture
```
├── 홈 화면
│   ├── 서비스 소개
│   ├── 가입
│   │   ├── 이메일 입력

│       ├── 문의하기
│       └── 서비스 이용 약관
├── 사용자 대시보드
│   ├── 기록 추가
│   ├── 기록 관리
│   ├── 피드백 보기
│   └── 분석 결과
│       ├── 최근 기록
```

## 섹션 6. MVP 정의 & Phase별 로드맵

### MVP 핵심 기능 세트
1. **사용자 기록 입력** — 포함 이유: 사용자가 자신의 생각과 경험을 기록하는 기능은 서비스의 가장 기본적이고 필수적인 요소로, 자체적으로 무지를 발견하기 위한 출발점이 됨.
2. **인지 오류 분석** — 포함 이유: 입력된 기록을 바탕으로 인지적 오류를 분석하여 사용자가 자신의 사고 패턴을 인식하고 개선할 수 있게 하는 핵심 기능으로, 서비스의 가치 제안과 직접 연결됨.
3. **개인화된 피드백 제공** — 포함 이유: 사용자의 개별 기록에 기반하여 개인 맞춤형 피드백을 제공함으로써 사용자 경험을 극대화하고, 재사용 유도를 위한 필수 요소임.
4. **대시보드** — 포함 이유: 사용자가 자신의 기록 및 피드백을 직관적으로 확인할 수 있도록 하여, 서비스의 유용성과 지속 사용 가능성을 높이는 시각적 요소임.
5. **기록 관리** — 포함 이유: 사용자가 자신의 기록을 수정 및 삭제할 수 있도록 해 사용자 편의성을 높이며, 데이터 관리의 권한을 부여해 신뢰성을 증진.

### MVP 검증 지표 & 성공 기준

### 성공 지표
1. 태스크 완료율: 각 시나리오에서 사용자가 설정한 전 단계 모두 완료 비율.
2. 에러율: 사용자가 태스크 수행 중 경험한 오류 수 및 문제 발생 비율.
3. NPS (Net Promoter Score): 사용자가 친구에게 추천하고 싶은지에 대한 평가, 주기적으로 설문조사를 통해 측정.

## 섹션 11. Tech Lead 전달 사항

### Feasibility 검토 요청 항목
1. 데이터 분석 및 인지 오류 분석 기능을 구현하기 위한 데이터 구조 최적화 방안.
2. 개인화된 피드백 제공을 위한 알고리즘의 구현 가능성 및 성능 테스트 계획.
3. 결제 시스템 통합 시 예상되는 보안 이슈 및 해결 방안.

### 기술 구현 우선순위 논의 포인트
1. 사용자 기록 입력 및 분석 기능: 초기 MVP를 위해 필수적이며, 리소스 분배 상 우선 순위.
2. 대시보드 및 피드백 제공 기능: 사용자 경험을 높이기 위한 중요 요소로, 빠른 프로토타입 검증 후 진행.
3. 결제 시스템: 기술적 복잡성을 감안하여, 사용성 테스트 후 구현 시점을 조율 필요.

## 🗺 워크플로우 개요

```
[사용자 입력]
    ↓
[Step 0] 입력 검증 & 보완 질문
    ↓
[Step 1] 타겟 사용자 페르소나 도출
    ↓
[Step 2] 시장 분석 & Pain Points
    ↓
[Step 3] 경쟁사 분석
    ↓
[Step 4] USP 설계
    ↓
[Step 5] 비즈니스 모델(BM) 기획
    ↓
[Step 6] KPI & 비즈니스 목표 수립
    ↓
[Step 7] Risk & 시나리오 플래닝
    ↓

```

이 순서는 **절대 변경하지 않는다.** 어떤 상황에서도 Step 0 → 7 순서를 준수한다.

## 📋 단계별 상세 행동 지침

### Step 0. 입력 검증 & 보완 질문
**목적**: 분석에 필요한 최소 입력값 확보

**체크 항목**:
- [ ] 초기 컨셉이 명확히 서술되었는가?
- [ ] 타겟 유저 가설이 존재하는가?

**불완전할 경우**:
분석을 시작하지 않고 즉시 아래 질문 중 부족한 항목을 구체적으로 질문한다.

| 부족한 정보 | 질문 예시 |
|------------|-----------|
| 초기 컨셉 | "어떤 제품/서비스인지 2~3문장으로 설명해주세요." |
| 타겟 유저 | "주요 고객은 누구인가요? 연령, 직업, 상황을 구체적으로 알려주세요." |
| 시장 범위 | "한국 시장만인가요, 글로벌 확장도 고려하나요?" |
| 경쟁 환경 | "현재 알고 있는 경쟁 서비스가 있다면 알려주세요." |

**완료 조건**: 초기 컨셉 + 타겟 유저 최소 2가지 확보 시 Step 1 진행

---

### Step 1. 타겟 사용자 페르소나 도출
**목적**: 실제 돈을 낼 사람이 누구인지 구체화

**필수 실행**:
- JTBD(Jobs-to-be-Done) 프레임워크 적용
- 페르소나 2~3개 생성 (연령·직업·생활 패턴·디지털 행동 포함)
- Pain Point 4~6개 도출 (감정적·기능적·사회적 분류)
- 각 Pain Point를 실제 사례나 숫자로 구체화

**체크포인트** (이 조건을 충족해야 다음 단계 진행):
- [ ] 페르소나가 2개 이상 작성되었는가?
- [ ] 각 페르소나에 JTBD 기반 동기가 명시되었는가?
- [ ] Pain Point가 4개 이상 도출되었는가?
- [ ] 모든 Pain Point에 감정적/기능적/사회적 분류가 붙었는가?

---

### Step 2. 시장 분석 & Pain Points
**목적**: 이 사업이 싸울 시장의 규모와 현실을 파악

**필수 실행**:
- TAM / SAM / SOM 숫자로 산정 (근거 명시 필수)
- 한국 시장 + 글로벌 트렌드 동시 분석
- PESTLE 분석으로 외부 환경 파악
- SWOT 분석으로 기회와 위협 도출
- 최신 시장 데이터 또는 유사 사례 인용

**체크포인트**:
- [ ] TAM/SAM/SOM이 구체적인 숫자로 제시되었는가?
- [ ] 근거(출처 또는 계산 근거)가 명시되었는가?
- [ ] SWOT 4개 항목 모두 작성되었는가?
- [ ] 한국 시장 특수성(규제, 소비자 행동)이 반영되었는가?

---

### Step 3. 경쟁사 분석
**목적**: 우리가 이길 수 있는 공간을 찾아내기

**필수 실행**:
- 직접 경쟁사 3개 + 간접 경쟁사 2개 + 대체재 1~2개 포함
- 경쟁사별 강점·약점·BM·가격 전략을 테이블로 정리
- Value Curve 또는 Blue Ocean Strategy로 차별화 포인트 시각화
- "우리만 할 수 있는 것" 명확히 도출

**체크포인트**:
- [ ] 직접 3 + 간접 2 + 대체재 최소 1개가 포함되었는가?
- [ ] 경쟁사 테이블에 강점/약점/BM/가격 전략이 모두 있는가?
- [ ] Value Curve로 차별화 포인트가 시각적으로 표현되었는가?
- [ ] "우리만 할 수 있는 것"이 1개 이상 명확히 도출되었는가?

---

### Step 4. USP 설계
**목적**: 고객이 우리를 선택해야 하는 이유를 3문장으로 압축

**필수 실행**:
- USP를 3문장 이내로 명확히 작성
- Value Curve 기반 차별화 곡선 설명
- 고객 관점에서 "왜 우리를 선택해야 하는가" 3가지 이상 제시

**체크포인트**:
- [ ] USP가 3문장 이내로 작성되었는가?
- [ ] 고객 선택 이유가 3가지 이상 구체적으로 제시되었는가?
- [ ] 모호한 표현("혁신적", "최고의" 등)이 없는가?

---

### Step 5. 비즈니스 모델(BM) 기획
**목적**: 실제로 돈이 어디서 어떻게 들어오는지 설계

**필수 실행**:
- 3가지 BM 옵션 제시 (Best Case / Realistic Case / Conservative Case)
- Business Model Canvas 9블록 전체 작성
- 가장 현실적인 수익 모델 1순위 추천
- Unit Economics 계산: CAC, LTV, Payback Period
- 가격 전략 구체적 예시 포함 (Tiered Pricing, Usage-based 등)

**체크포인트**:
- [ ] BM 옵션이 3가지 제시되었는가?
- [ ] Business Model Canvas 9블록이 모두 작성되었는가?
- [ ] CAC, LTV, Payback Period가 계산되었는가?
- [ ] 가격 전략에 구체적인 금액 예시가 포함되었는가?

---

### Step 6. KPI & 비즈니스 목표 수립
**목적**: 성공의 기준을 숫자로 정의

**필수 실행**:
- North Star Metric 1개 선정
- Leading Indicator 3개 + Lagging Indicator 3개 설정
- 1년 목표와 3년 목표를 구체적인 숫자로 제시
- OKR 형식으로 정리

**체크포인트**:
- [ ] North Star Metric이 1개 명확히 선정되었는가?
- [ ] Leading 3개 + Lagging 3개가 모두 작성되었는가?
- [ ] 1년/3년 목표가 구체적인 숫자로 제시되었는가?
- [ ] OKR 형식으로 정리되었는가?

---

### Step 7. Risk & 시나리오 플래닝
**목적**: 실패 시나리오를 미리 준비하여 생존 가능성 높이기

**필수 실행**:
- 주요 리스크 5~7개 식별
- 발생 확률 × 영향도 매트릭스 작성
- 각 리스크별 Mitigation Plan + Contingency Plan 제시
- Best / Base / Worst Case 3년 재무 전망 표 작성

**체크포인트**:
- [ ] 리스크가 5개 이상 식별되었는가?
- [ ] 발생 확률과 영향도가 매트릭스로 정리되었는가?
- [ ] 모든 리스크에 Mitigation + Contingency Plan이 있는가?
- [ ] 3년 재무 전망 표가 Best/Base/Worst로 구분되어 있는가?

## ✅ 완료 후 필수 실행

### decisions.md 기록
모든 Step이 완료되면 **즉시** `docs/decisions.md`를 열고:
1. 분석 보고서 템플릿을 복사
2. 해당 날짜 + 프로젝트명으로 새 섹션 추가
3. 분석 결과를 빠짐없이 채워서 기록
4. 가장 최근 분석이 최상단에 오도록 배치

### progress.md 업데이트
완료한 항목을 `[ ]` → `[x]`로 변경하고 진행률 재계산
